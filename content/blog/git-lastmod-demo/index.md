这是一篇本地测试文章，用来确认我们现在的写作模板已经切到了更简单的版本。

## 这篇文章主要验证什么

- `date` 会作为创建时间保留下来
- `lastmod` 不再手写，而是交给 Git 自动提供
- 新文章模板不再默认塞一堆空字段

## 现在的最小字段长什么样

现在一篇新文章的最小 front matter 是：

```yaml
---
title: "你的文章标题"
date: 2026-05-04T16:25:00+08:00
draft: true
---
```

如果你需要分类或标签，再自己加：

```yaml
categories: ["测试"]
tags: ["Git", "Hugo"]
```

## 什么时候会看到 lastmod 更新

`lastmod` 现在不是靠你手写，而是优先读取 Git 的最后一次提交时间。

也就是说：

1. 你修改文章内容
2. 提交 Git
3. Hugo 构建时读取这篇文件最新的 Git 提交时间
4. 页面里显示的更新时间就跟着变

## 这样做的好处

- 创建时间和更新时间职责更清楚
- front matter 更干净
- 写文章的时候不需要一直盯着一堆元字段

## 结尾

如果这篇文章在本地能正常显示，说明我们现在的文章模板已经适合正式开始写内容了。
